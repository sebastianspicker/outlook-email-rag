# Document Request And Preservation Checklist

## Goal

Convert missing-proof analysis into a practical records-request and preservation workflow.

## Output

`document_request_checklist`

Fields:

- `version`
- `group_count`
- `groups`

Each group contains:

- `group_id`
- `title`
- `item_count`
- `items`

Each item contains:

- `item_id`
- `request`
- `why_it_matters`
- `likely_custodian`
- `would_prove_or_disprove`
- `urgency`
- `risk_of_loss`
- `preservation_action`
- `linked_date_gap_ids`

## Intended request groups

- personnel file
- home office or mobile work documents
- Tätigkeitsdarstellung and job evaluation records
- task-change communications
- SBV records
- PR / LPVG records
- BEM / prevention records
- accommodation records
- time system or attendance records
- calendar invites or meeting notes
- comparator evidence

## Rules

- Items stay document-oriented and custodian-aware.
- Urgency and loss risk stay explicit.
- Preservation actions should identify how to keep native records and metadata.
- The checklist may combine missing-exhibit, chronology-gap, and skeptical-review repair signals.
